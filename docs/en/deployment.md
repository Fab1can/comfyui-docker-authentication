# Deployment Guide

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) ≥ 24
- [Docker Compose](https://docs.docker.com/compose/) ≥ 2
- [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html) (required for ComfyUI GPU support)

---

## 1. Clone the repository

```bash
git clone https://github.com/Fab1can/comfyui-docker-authentication.git
cd comfyui-docker-authentication
```

---

## 2. Configure environment variables

Copy the example file and edit as needed:

```bash
cp auth-server/.env.example auth-server/.env
```

At minimum, set a strong `JWT_SECRET`:

```ini
# auth-server/.env
JWT_SECRET=replace-with-a-long-random-secret
```

Optionally, change the host port in the root `.env`:

```ini
# .env
HOST_PORT=8188
```

---

## 3. Start the stack

```bash
docker compose up -d
```

All three services (`comfyui`, `auth-server`, `nginx`) will start.  
On first run, Docker will pull images and build the auth-server image.

Check that all containers are healthy:

```bash
docker compose ps
```

---

## 4. Get the current OTP

```bash
docker compose logs auth-server | grep -E "OTP"
```

The OTP rotates every 30 seconds (configurable). Copy the most recent value.

---

## 5. Log in

Open a browser and navigate to:

```
http://localhost:8188/auth/login?otp=<OTP>
```

Or use `curl`:

```bash
curl -c cookies.txt "http://localhost:8188/auth/login?otp=<OTP>"
```

A successful response sets the `jwt` HTTP-only cookie.

---

## 6. Open ComfyUI

Navigate to `http://localhost:8188` in your browser.  
The cookie is sent automatically and Nginx grants access to ComfyUI.

---

## Stopping the stack

```bash
docker compose down
```

To also remove the workspace volume:

```bash
docker compose down -v
```

---

## Updating

```bash
docker compose pull          # update ComfyUI and Nginx images
docker compose build         # rebuild auth-server image
docker compose up -d
```

---

## Troubleshooting

### `401 Unauthorized` on every request

- Verify the `jwt` cookie is present in the browser's DevTools → Application → Cookies.
- Check that `JWT_SECRET` is the same in `docker-compose.yml` and any `.env` file.
- Confirm `auth-server` is running: `docker compose ps`.

### `502 Bad Gateway`

- ComfyUI may still be loading models. Wait a few seconds and refresh.
- Check ComfyUI logs: `docker compose logs comfyui`.

### OTP not visible in logs

```bash
docker compose logs --tail=50 auth-server
```

Look for lines such as:
```
New OTP generated: 57312
```

### Container fails to start — GPU error

Ensure the NVIDIA Container Toolkit is installed and Docker is configured to use it:

```bash
docker run --rm --gpus all nvidia/cuda:12.0-base nvidia-smi
```

### Rate-limited (`429 Too Many Requests`)

Wait for `LOCKOUT_DURATION` ms (default 10 minutes) before trying again, or restart the auth-server container to clear the in-memory attempt counter:

```bash
docker compose restart auth-server
```

## Production recommendations

| Concern | Recommendation |
|---|---|
| **HTTPS** | Place a TLS-terminating reverse proxy (e.g. Traefik, Caddy) in front of Nginx |
| **JWT_SECRET** | Use a random string of ≥ 32 characters generated by `openssl rand -hex 32` |
| **JWT_EXPIRATION** | Reduce to `1d` or `8h` for sensitive environments |
| **OTP delivery** | Use a log aggregation tool or alerting pipeline to deliver the OTP securely |
| **Secrets management** | Use Docker secrets or an external vault instead of plain-text `.env` files |
